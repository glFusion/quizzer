Custom Form Field Definitions

Field Definitions
Field Name This is the name of the custom field used in the database. It must be unique to this form.
Text Prompt This is the text that will be displayed next to the fields in forms and display screens. For example, a custom field named "birthdate" might have a prompt of "Birth Date".
Field Type Select the type of field to be used. This selection will determin which of the remaining options are displayed and how they are used.
Length (Type=text) These values determind the display length and actual maximum length of entries that can be made.
Max. Length (Type=text) Enter the maximum number of characters that may be entered into this field.
Columns (Type=textarea) Enter the number of columns for the textarea field.
Rows (Type=textarea) Enter the number of rows for the textarea field.
Calculation Type (Type=calculation) Select the type of calculation to be performed- Addition, Subtraction, etc.
Default Value
  • (Type=text, radio, checkbox)
    Enter the default value for this field. For text fields, this is a simple string. For radio buttons, enter the value corresponding to the option to be selected as default. For checboxes, enter either 1 or 0.
     
  • (Text fields only)
    You can enter a default value as "$_USER:varname", where "varname" is a variable from the $_USER array, such as "email" or "fullname."
     
  • (Type=static) Enter the HTML or static text that you'd like to appear. Leave this field empty to have a blank line inserted into the form.
     
  • (Type=date) Enter the default date as "YYYY-MM-DD". If the "Show Time" box is checked, you can add the time as "HH:MM" (24-hour format).
    The special string "$NOW" may be entered to indicate that the field should default to the current date and time.
     
  • (Type=time) Enter the default time as "HH:MM" (24-hour format).
    The special string "$NOW" may be entered to indicate that the field should default to the current time.
Value Enter the values to be used for selections, etc. The usage depends upon the type of field being defined.
  • (Type=radio,dropdown,multicheck)
    Create the values to be used in the radio buttons. Click the "Add" icon to add a new value to the array. Click the radio button next to the value that should be selected by default.
     
  • (Type=calculation)
    Enter the values or field names to be used in the calculation. Field names must exactly match the "Name" parameter for the fields used; if not, the field will be ignored. Numeric values will be treated literally.
     
    Example: if the field type is "Addition" and the value is "2,6,amount", then the result would be 2 + 6 + (numeric value of the "amount" field). If a field value is non-numeric then it is ignored. For division, a value of zero for other than the first value is also ignored.
     
  • (Type=static)
    Enter the static text that should appear on the form. You can include a value from the $_USER array as "$_USER:varname", where "varname" is a variable from the array such as "email" or "fullname."
     
Format (Type=calculation) Enter an optional numeric format to be used when displaying the results. If this is blank, "%8.2f" will be used. Refer to the PHP documentation for the sprintf() function for formatting options.
Auto-Generate? Check this box if this field should be automatically generated. The plugin will generate a value for this field using the first method that is available:
  1. You can define custom function in lib-custom.php named "CUSTOM_forms_autogen_{fieldname}" to create a function specifically for this field.
  2. You may also create "CUSTOM_forms_autogen()" to create a generic auto-generation function for all forms.
  3. If you do neither, then the plugin will create a unique value using COM_makeSid().
Field Mask (TESTING) (Type=text) Enter the mask string to be used with iMask for this field.
  • "9" represents a digit.
  • "A" represents a letter.
  • "X" represents an alphanumeric character.
Show Time? (Type=date) Check this box if the time selector is to be available on the datepicker.
12 or 24-hour format? (Type=date) Select whether times will be entered in 12-hour or 24-hour format.
Add century if missing? (Type=date) Check this box to have the current century added to values of less than 100. Leave unchecked if you expect very small date values to be entered, in which case the user input will be taken literally.
Format (Type=date) Enter the format string to be used for the date/time. Refer to the PHP date formats for more information.
Value (Type=radio, dropdown) This is a string which indicates the possible values for dropdown selections and radio buttons. The format is "value:text;value:text;..." For example, to have two radio buttons labeled "Red" and "Blue", which return values of "R" and "B" respectively, you would enter "R:Red;B:Blue" as the value.
Enabled Check this box to enable the field, uncheck to disable. This allows the administrator to prevent some fields from being used without deleting them. (Deleting a custom field also deletes all user data associated with it).
Span All Columns? Check this box to have the field's entry area span all columns of the form. This causes the prompt to be supressed and the field will be left-aligned. If you only want to omit the text prompt, but have the field aligned with the others, then leave the prompt empty.
Help Text Enter an optional short help message to be displayed as a tooltip after the entry field.
Group that can fill out field Select the user group that will be allowed to see and fill out this field. This allows some fields to be presented to only certain users.
Group that can see results Select the user group that can see this field in the results table. This controls the results display via the "forms:results" autotag, not in the administrator's control panel.
Field Access Select the access level for this field. The options are:
  • Normal - Users with the appropriate access level can enter data into this field.
  • Required - This field requires something to be entered. Selecting this option for a Checkbox-type field means that the box must be selected before the form can be submitted. This may be useful for acknowledging a "terms and conditions" page, for example.
  • Read-Only - The field is displayed but cannot be edited.
  • Hidden - The field is hidden from all users.
The Read-Only and Hidden options should be used only when the field is auto-generating or has a useful default value.
Position After Select the existing field after which this field should be displayed on the form. The default for new fields is to become the last field on the form.
