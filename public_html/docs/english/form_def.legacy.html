Custom Form Definitions

Form Attributes
Form ID This is the unique ID of the form. It should be short and a single word (no spaces) to be used in autotags.
Form Name This is a short descriptive name for the form, such as "Event Signup". It does not have to be unique, but should be to avoid confusion.
Intro Text Enter some text or instructions to be displayed before the forms' fields.
Message after Submission *Not Implemented* Enter some text to be displayed in a popup message after the form is submitted. This function is currently under development; for now a static message is defined in the language file.
Message shown if user can't access the form Enter a message to be displayed to users if they have no access to the form. This is optional; if nothing is entered here then nothing at all will be displayed. HTML is allowed.
Redirect URL Eneter a destination URL to redirect the submitter after submission. Substitute "{site_url}" for your site's base URL.
Enabled Check this box if this form is to be enabled. This provides a quick way to prevent forms from being used without deleting them.
Action upon Submission When submitted, forms may have their data added to the database, emailed to the form's owner, or both. Select which actions you want to occur.
Moderate? *Not Implemented* Check this box if user submissions must be approved before they become visible in the results table or for download.
Max. Submissions Enter a positive integer here if there should be a limit on the total number of submissions. If this field is empty or zero, then unlimited submissions are accepted.
Owner Select the user to become the "owner" of this form. The owner can see and download all results.
Admin Group Select a group to be the Admin group for this form. The admin group has all rights that the owner has.
Group that can fill out form Select the user group that is allowed to fill out this form. If the current user is not a member of this group, they will not have access to the form.
Group that can see results Select the user group that will be able to see the results for this form. When results are shown in an autotag, users must be a member of this group or nothing will be displayed. For example, you may not want regular users to see other members who will be attending an event, or other bids on items.
CAPTCHA Check this box to require a CAPTCHA to be entered along with the form. The global site configuration governs whether the CAPTCHA is shown to all users or only to Anonymous.
Reset Field Permissions Check this box to reset all field permissions to match the form's permission settings.
