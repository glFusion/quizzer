glFusion Forms Plugin - Field Definition
glFusion Wiki

glFusion Profile Plugin Configuration

Field Name
This is the name of the custom field used in the database. It must be unique to this form.
Text Prompt
This is the text that will be displayed next to the fields in forms and display screens. For example, a custom field named "birthdate" might have a prompt of "Birth Date".
Field Type
Select the type of field to be used. This selection will determin which of the remaining options are displayed and how they are used.
Length
These values determind the display length and actual maximum length of entries that can be made.
Max. Length
Enter the maximum number of characters that may be entered into this field.
Columns
Enter the number of columns for the textarea field.
Rows
Enter the number of rows for the textarea field.
Calculation Type
Select the type of calculation to be performed- Addition, Subtraction, etc.
Default Value
Enter or select the default value for this field.
Value
Enter the values to be used for selections, etc. The usage depends upon the type of field being defined.
Format
Enter an optional numeric format to be used when displaying the results. If this is blank, "%8.2f" will be used. Refer to the PHP documentation for the sprintf() function for formatting options.
Auto-Generate?
Check this box if this field should be automatically generated. The plugin will generate a value for this field using the first method that is available from CUSTOM_forms_autogen(fieldname), CUSTOM_forms_autogen(), or COM_makeSid()
Field Mask (TESTING)
Enter the mask string to be used with iMask for this field. "9" represents a digit, "A" represents a letter and "X" represents an alphanumeric character. Any other character is displayed as entered.
Show Time?
Check this box if the time selector is to be available on the datepicker.
12 or 24-hour format?
Select whether times will be entered in 12-hour or 24-hour format.
Add century if missing?
Check this box to have the current century added to values of less than 100. Leave unchecked if you expect very small date values to be entered, in which case the user input will be taken literally.
Format
Enter the format string to be used for the date/time. Refer to the PHP date formats for more information.
Date Input Format
Select the format for date inputs.
Value
This is a string which indicates the possible values for dropdown selections and radio buttons. The format is "value:text;value:text;..." For example, to have two radio buttons labeled "Red" and "Blue", which return values of "R" and "B" respectively, you would enter "R:Red;B:Blue" as the value.
Enabled
Check this box to enable the field, uncheck to disable. This allows the administrator to prevent some fields from being used without deleting them. (Deleting a custom field also deletes all user data associated with it).
Span All Columns?
Check this box to have the field's entry area span all columns of the form. This causes the prompt to be supressed and the field will be left-aligned. If you only want to omit the text prompt, but have the field aligned with the others, then leave the prompt empty.
Help Text
Enter an optional short help message to be displayed as a tooltip after the entry field.
Group that can fill out field
Select the user group that will be allowed to see and fill out this field. This allows some fields to be presented to only certain users.
Group that can see results
Select the user group that can see this field in the results table. This controls the results display via the "forms:results" autotag, not in the administrator's control panel.
Field Access
Select the access level for this field. The Read-Only and Hidden options should be used only when the field is auto-generating or has a useful default value.
Position After
Select the existing field after which this field should be displayed on the form. The default for new fields is to become the last field on the form.
